Wrangle Report

A wrangling project typically has three processes. Gathering, Assessing and Cleaning. As part of my Udacity project, I was able to successfully complete all three processes to create a cleaned master set of the WeRateDogs data.

Gathering

At the gathering stage, you are tasked to collect data from different sources and load them into dataframes in pandas. You can get this data from files, web scraping or querying API's.In this stage I was able to perform the following;

  • Load CSV file into dataframe using pandas
  • Download file programmatically using requests
    • Query Twitter API and read data from JSON file line by line into the dataframe

Assessing

This is the stage where you access the data and determine dirtyness(quality) or tidiness issues that it posesses which may interrupt your work. There are two methods of assessing data;

  • Visually - Examining the dataset manually and spotting various issues
  • Programmatically - Querying the dataset with code to spot underlying issues

There are two groups of issues your assessment may reveal: Quality and Tidiness Issues. I was able to discover the following quality and tidiness issues;

Quality Issues

  1. Timestamp and create_date columns in archive_df and tweet_data are strings
  2. Hard coding of missing dog type and dog name columns entries to "None"
  3. HTML tags in source column in archived_df
  4. Missing entries in expanded_url column of archive_df
  5. Hard coding of missing dog type and dog name columns entries to "None"
  6. Double values in expanded_url column.
  7. Retweets or replies need to be removed from the dataframe archive_df
  8. Tweet ID in the dataframes are integers
  9. Some columns have the wrong rating

Tidiness Issues

  1. Dog type (e.g. puppo) should be in one column
  2. Retweet count and favourite count should be in archive_df dataframe

Cleaning

At this stage, all the issues identified in the assessment stage are resolved allowing us to have a final, quality and tidy dataset.
